# Claude + ClawX 协作协议

## 能力矩阵

| 能力 | Claude（我） | ClawX（你） |
|---|---|---|
| 看图片/截图 | ✅ 可以 | ✅ 更强（实时桌面） |
| 操控浏览器 | ✅ Chrome MCP | ✅ 原生操控 |
| 读写文件 | ✅ E盘共享区 | ✅ 本地所有盘 |
| 操控桌面App | ❌ 不能 | ✅ 核心能力 |
| 写代码 | ✅ 主攻 | ✅ 可以 |
| 数据分析 | ✅ 主攻 | ✅ 可以 |
| 规划/设计 | ✅ 主攻 | ⚠️ 一般 |
| 本地系统操作 | ❌ 沙盒限制 | ✅ 无限制 |

## 分工原则

**我更适合：**
- 写代码、改网站、生成内容（一次产出几百行）
- 分析数据、做竞品报告、制定策略
- 搜索网页、总结信息
- 规划多步骤任务的执行方案

**ClawX 更适合：**
- 在本地电脑上安装/配置软件
- 操控桌面 App（微信、浏览器、文件管理器）
- 实时监控屏幕上的变化（比如盯盘、看页面渲染效果）
- 需要系统权限的操作（注册表、进程管理、外设）

**配合模式：**
1. 我写任务文件 → E:\ai导航网站\clawx-task.json
2. ClawX 读取 → 执行 → 结果写回 clawx-reply.json
3. 我读到结果 → 继续下一步

## 任务格式

```json
{
  "task": "任务名",
  "type": "screenshot|file-op|browser|app-control|system",
  "instructions": "具体做什么",
  "output": "结果写到哪里"
}
```

## 典型协作场景

**场景A：我写网站 → ClawX 验证效果**
我改完代码 → 写任务让 ClawX 打开 localhost:3000 → 截图 → 我读图确认

**场景B：ClawX 抓数据 → 我分析**
ClawX 打开竞品定价页 → 截图/复制文字 → 存 E盘 → 我读数据做分析

**场景C：我规划 → ClawX 在服务器上执行**
我列部署步骤 → ClawX SSH 连服务器 → 逐步执行 → 每步反馈结果

**场景D：ClawX 盯盘 → 我报警**
ClawX 定时截图行情 → 我分析是否有异动 → 有异动通知用户
